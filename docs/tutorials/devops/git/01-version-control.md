# Git Version Control

## Introduction
Git is the standard for version control. It enables collaboration and infrastructure-as-code.

## Core Commands
- `git init`: Initialize repo.
- `git clone`: Copy a repo.
- `git add` / `git commit`: Stage and save changes.
- `git push` / `git pull`: Sync with remote.
- `git status`: Check state.

## Branching Strategies
- **Feature Branching**: Create a new branch for every new feature or fix.
- **Pull Requests**: Review code before merging to main.
