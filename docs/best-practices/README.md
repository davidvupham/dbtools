## What Belongs Here

**Best Practices** are prescriptive standards that define *how* things should be done across the organization:

- **Project management standards** - Documentation frameworks, approval workflows
- **Coding standards** - Style guides, naming conventions (see also: [development/coding-standards/](../development/coding-standards/))
- **Operational standards** - Monitoring, alerting, incident response patterns
- **Security standards** - Authentication, authorization, secrets management patterns

## How Best Practices Differ from Other Docs

| Documentation Type | Purpose | Example |
|--------------------|---------|---------|
| **Best Practices** | Standards to follow | "All projects must have a decision log" |
| **How-to Guides** | Solve a specific problem | "How to rotate AD passwords" |
| **Tutorials** | Learn a technology | "Introduction to Vault" |
| **Reference** | Technical specifications | "Vault CLI command reference" |
