## Course description

This comprehensive course teaches Markdown from the ground up, progressing from basic syntax to advanced features and professional tooling. You'll learn to write documentation, README files, technical articles, and collaborate effectively using Markdown across various platforms.

## Learning objectives

By completing this course, you will be able to:

1. **Write well-formatted documents** using core Markdown syntax
2. **Create professional README files** for software projects
3. **Use GitHub Flavored Markdown** features including tables, task lists, and alerts
4. **Generate diagrams** using Mermaid syntax directly in Markdown
5. **Write mathematical equations** using LaTeX/KaTeX notation
6. **Convert documents** between formats using Pandoc
7. **Integrate Markdown** into documentation workflows and static site generators

## Target audience

- Developers writing documentation and README files
- Technical writers adopting lightweight markup
- Students and academics writing papers and notes
- Anyone transitioning from word processors to plain text
- Teams collaborating on documentation in version control

## Prerequisites

- Basic computer literacy
- A text editor (any will work; VS Code recommended)
- For Part 3: Basic command line familiarity

## Course structure

| Part | Title | Level | Topics |
|------|-------|-------|--------|
| **1** | [Markdown basics](./learning-paths/series-part1-basics.md) | Beginner | Headings, text formatting, links, images, lists, code, blockquotes |
| **2** | [Intermediate features](./learning-paths/series-part2-intermediate.md) | Intermediate | Tables, task lists, footnotes, alerts, emoji, escaping, GFM |
| **3** | [Advanced techniques](./learning-paths/series-part3-advanced.md) | Advanced | Mermaid diagrams, math, HTML, Pandoc, documentation frameworks |

## What you'll build

Throughout this course, you'll create practical documents:

- **Part 1**: A project README file with proper formatting
- **Part 2**: A technical specification with tables and task lists
- **Part 3**: A documentation site with diagrams and automated conversion

## Technology stack

- **Core Syntax**: CommonMark specification
- **Extended Syntax**: GitHub Flavored Markdown (GFM)
- **Diagrams**: Mermaid, PlantUML
- **Math**: KaTeX, LaTeX
- **Conversion**: Pandoc
- **Editors**: VS Code, Obsidian, Typora (your choice)

## Markdown flavors covered

| Flavor | Description | Used By |
|--------|-------------|---------|
| **CommonMark** | Standard specification | Base for all flavors |
| **GFM** | GitHub Flavored Markdown | GitHub, GitLab |
| **MultiMarkdown** | Extended features | Academic writing |
| **R Markdown** | With R code execution | Data science |

## How to practice

1. **Interactive exercises**: Use [markdowntutorial.com](https://www.markdowntutorial.com/) alongside this course
2. **Live preview**: Use VS Code with the Markdown Preview Enhanced extension
3. **GitHub**: Create a repository and practice with README files
4. **Local files**: Create `.md` files and view them in your editor
