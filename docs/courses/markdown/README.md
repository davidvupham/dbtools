## Introduction

This is a **comprehensive tutorial series** on Markdown, taking you from complete beginner to advanced user. Markdown is a lightweight markup language that has become the standard for technical documentation, README files, wikis, and collaborative writing across platforms like GitHub, GitLab, Confluence, and static site generators.

### Why learn Markdown?

- **Universal adoption**: Used by GitHub, GitLab, Reddit, Stack Overflow, Discord, Slack, and countless other platforms
- **Plain text simplicity**: Write once, render anywhere without proprietary formats
- **Version control friendly**: Perfect for Git-based workflows
- **Focus on content**: Write faster without fighting formatting tools
- **Portable**: Your documents work across any operating system or application

## Learning paths

Choose the path that fits your experience level.

### Recommended path (Beginner to advanced)

*Best for: Users new to Markdown or those wanting a solid foundation.*

1. **[Part 1: Markdown basics](./learning-paths/series-part1-basics.md)**
   - *Goal*: Master core Markdown syntax for everyday writing.
   - *Topics*: Headings, paragraphs, emphasis, links, images, lists, code blocks, blockquotes.

2. **[Part 2: Intermediate features](./learning-paths/series-part2-intermediate.md)**
   - *Goal*: Learn extended syntax and GitHub Flavored Markdown (GFM).
   - *Topics*: Tables, task lists, footnotes, definition lists, alerts/admonitions, emoji, escaping.

3. **[Part 3: Advanced techniques](./learning-paths/series-part3-advanced.md)**
   - *Goal*: Master advanced features, tooling, and professional workflows.
   - *Topics*: Mermaid diagrams, math equations, HTML embedding, Pandoc conversion, documentation frameworks.

## Frequently asked questions

### Q: Do I need to complete Part 1 before Part 2?

**A:** Recommended but not required. If you already know basic Markdown syntax (headings, links, lists), you can start at Part 2. Part 3 assumes familiarity with Parts 1 and 2.

### Q: Which Markdown flavor should I learn?

**A:** Start with CommonMark (standard Markdown), then learn GitHub Flavored Markdown (GFM) as it's the most widely used extension. This course covers both.

### Q: Can I convert my existing documents to Markdown?

**A:** Yes. [Pandoc](https://pandoc.org/) can convert Word, PDF, HTML, Confluence, and many other formats to Markdown. See Part 3 for details.

### Q: What editor should I use?

**A:** Any text editor works. Popular choices include VS Code (with Markdown extensions), Obsidian, Typora, and online editors like StackEdit or Dillinger.
