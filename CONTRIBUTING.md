# Contributing to dbtools

Thank you for your interest in contributing to `dbtools`!

## Getting Started

We have detailed documentation available to help you set up your environment and understand our standards.

- **[Development Guide](docs/development/README.md)**: The main index for all development topics.
- **[Setting up VS Code](docs/development/vscode/README.md)**: Instructions for configuring VS Code and Dev Containers.

## Coding Standards

Please ensure your code leads strictly to our defined standards:

- **[Python Standards](docs/development/coding-standards/python-coding-standards.md)**
- **[PowerShell Standards](docs/development/coding-standards/powershell-coding-standards.md)**
- **[General Standards](docs/development/coding-standards/language-agnostic-coding-standards.md)**

## Testing

All changes must be verified with tests:

- **[Testing Guide](docs/development/testing/README.md)**: Comprehensive testing strategy and instructions.
- **[Quick Reference](docs/development/testing/quick-ref.md)**: Cheat sheet for common test commands.

## Submitting Changes

1.  Ensure you have run the appropriate build/test scripts (e.g., `./build.ps1` or `make all`).
2.  Submit a Pull Request targeting the `main` branch.
